1. Crear modelo entidad relación

Un modelo de entidad relación es la estructura de las tablas que nos darán vida a la sabe de datos, nos ayuda a identificar tablas, datos, llaves primarias, llaves foráneas, las uniones entre tablas.


Para crear un modelo de entidad relación primero debemos trabajar con las formas normales, estas formas normales se usan en bases de datos relacionales para organizar la información correctamente y evitar problemas como datos repetidos, inconsistencias o anomalías al actualizar.


Su objetivo principal es normalizar una base de datos, es decir, estructurar las tablas de forma lógica y eficiente.

Consultas SQL Intermedias

1. Mostrar todas las facturas con sus clientes:
SELECT f.idfactura, c.nombres_cliente, f.Cliente_idCliente
FROM factura f
JOIN cliente c ON f.Cliente_idCliente = c.idCliente;

2. Mostrar detalle de una factura:
SELECT
f.idfactura,
c.nombres_cliente,
v.nombres_vendedor,
m.efectivo,
m.tarjeta
FROM factura f
JOIN cliente c ON f.Cliente_idCliente = c.idCliente
JOIN vendedor v ON f.Vendedor_idvendedor = v.idvendedor
JOIN metodo_pago m ON f.Metodo_pago_idMetodo_pago = m.idMetodo_pago
WHERE f.idfactura = 2;

Consultas SQL Avanzadas

1. Calcular las ventas totales:
SELECT SUM(CAST(precio AS DECIMAL)) AS ventas_totales
FROM productos;

2. Productos mas vendidos:
SELECT p.nombre_productos, SUM(s.cantidad) AS cantidad_vendida
FROM productos p
JOIN stock s ON p.stock_idstock = s.idstock
GROUP BY p.nombre_productos
ORDER BY cantidad_vendida DESC
LIMIT 1;

3. Cliente con mayor gasto:
SELECT c.nombres_Cliente, SUM(CAST(p.precio AS DECIMAL)) AS gasto_total
FROM cliente c
JOIN factura f ON c.idCliente = f.Cliente_idCliente
JOIN productos p ON f.idfactura = p.factura_idfactura
GROUP BY c.nombres_Cliente
ORDER BY gasto_total DESC;

4. Actualizar stock después de una venta:
UPDATE stock
SET cantidad = cantidad - 20
WHERE idstock = (SELECT stock_idstock FROM productos WHERE idProductos = 2);
